Git Übung

Ich habe ein neues Repository auf Github erstellt. Dann bin ich im Terminal auf den dev Ordner gegangen und habe das Repository geklont.

  • dev cd test
  • test git:(master) git branch testseite
  • test git:(master) git checkout testseite
  • u Branch 'testseite' gewechselt
  • test git:(testseite) code .

In VSCode habe ich über Ctrl+Shift+`das Terminal geöffnet und hier mit dem Touch Command eine index.html angelegt. Nachdem ich nun diese Datei angelegt habe, kann ich sie adden, committen und pushen.

zweite Branch

Nachdem ich die Testseite gepushed habe, habe ich eine Pull Request erstellt und die Testseite mit dem Master gemerged. Dann habe ich mir im Terminal die aktuellste Version heruntergeladen:

  • git checkout master
  • git pull
  • git branch
  • git branch -d testseite

Mit git checkout -b seite 2 habe ich mir eine neue Branch erstellt und diese direkt geöffnet. Nun kann ich meine neuen Änderungen wieder adden, committen und pushen.

Nach dem Pushen kann ich noch weitere Änderungen vornehmen und einen zweiten Commit machen.

Beim ersten Push muss ich den Namen der Branch angeben git push - u origin Name. Ab dem zweiten Push reicht dann git push. Außerdem reicht es bei einer bestehenden Datei den command git commit -am " " zu verwenden. Vorausgesetzt, man möchte alles adden und committen. Somit spart man sich die zwei separaten Schritte.

Konflikte beim Mergen

Wenn mehrere Leute gleichzeitig an verschiedenen Branches arbeiten, kann es beim Mergen zu Konflikten kommen, wenn z.B. verschiedene Änderungen in der gleichen Zeile gemacht wurden. Um diesen Konflikt zu lösen:

  1. git checkout master
  2. git pull
  3. git checkout branch
  4. git merge master
    5. VSCode zeigt dann die Änderungen an und bietet die Möglichkeit diese zu akzeptieren oder abzulehnen.
    1. Speichern
    2. add + commit + push
      3. Nun sollte man die Dateien in Github mergen können.
